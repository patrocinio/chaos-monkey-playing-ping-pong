Repo structure: 

Kubernetes templates:
Game-1/ping_pong/src/ping.yaml - kind: pod. Creates pod named ping. container image: patrocinio/ping_pong. 
Game-1/ping_pong/src/pong.yaml  - same as pong.yaml. Creates pod named pong.
Game-1/commentator/src/commentator_rset.yaml  - kind: Replicaset. Container image - ping_pong_commentator.
Game-1/throw_ball/deploy/throw_ball_job.yaml - kind: Job. Container image: patrocinio/ping_pong_throw_ball.

Python programs:
Game-1/ping_pong/src/message.py:
Ivokes ```Game-1/common/src/message.py```
Game-1/ping_pong/src/ping_pong.py:
Heart of the game. Initialize queue by consuming first ball on the ping queue. Counts ball. Publishes to the pong queue. (who calls it? - throwball?)
Game-1/ping_pong/src/pp_queue.py:
Invokes ```common/src/queue.py```
Game-1/commentator/src/commentator.py:
Calls ```Game-1/common/src/pp_queue.py ``` with message "Welcome to the Ping Pong solution”. Also it prints message as stdout.
Game-1/commentator/src/pp_queue.py:
Calls “../../common/src/queue.py” which is wrong. I should be “./../common/src/pp_queue.py”
Game-1/common/src/message.py:
Creates a new queue 'message' and publish message to it. Where message is hostname + ": " + message provided by caller.
Game-1/common/src/pp_queue.py 
It’s a consumer. Transform queue host into env var.
Game-1/throw_ball/src/message.py:
Invokes Game-1/common/src/message.py
Game-1/throw_ball/src/pp_queue.py - consumer 
Game-1/throw_ball/src/throw_ball.py - throw ball to the ping queue.
Game-1/throw_ball.py - throw ball to the ping queue.

Shell scritps:
Game-1/commentator/build_commentator.sh - creates image ping_pong_commentator:latest
Game-1/ping_pong/build_ping_pong.sh  - create image ping_pong:latest
Several other scripts under Games/scripts/ - helm init etc.
